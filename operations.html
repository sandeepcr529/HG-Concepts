Operations

Command syntax

All mercurial commands start with hg and have syntax hg command arguments...

Creating a Repository

The command hg init creates a blank repo at the current directory. This means, if you are in directory /home/abc, then running hg init will create a directory /home/abc/.hg. At the beginning the repo will not contin any files, even though /home/abc may contain any files.

Adding files to Repository

The command hg add is used to add files to the repository.

If no file name is provided, then this will add ALL the files that are present in the current directory. A file is added to the repository means that file becomes a tracked file.

Commiting

Commit operation saves the state of the tracked files in the repo as a new changeset. The command for this operation is, hg commit -u username -m commit-messageThe username and message can be any string. If they contain spaces, then they have to be quoted like hg commit -u sandeep -m "added missing argument"

Going back in history

Suppose you have just made your 20th commit and for whatever reason you want to know what your project looked like when you made the 10th commit. This can be done in 2 ways

  • By Reverting
  • By Updating

Revert

hg revert -a -r 10 reverts all the tracked files in your working directory to the state they were when you made the 10th commit. The parent of the working directory is not changed. If the parent of the working directory was 20, then after reverting to 10th revision the parent of the working directory is still 20th revision. If you make a commit now, then the new revision will be added as a child of 20th revision.

Update

hg update -r 10 reverts all the tracked files in your working directory to the state they were when you made the 10th commit and changes the parent of the working directory to 10th revision. If the parent of the working directory was 20, then after updating to 10th revision the parent of the working directory is 10th revision. If you make a commit now, then the new revision will be added as a child of 10th revision. If you do this, you will create an anonymous branch.
